Google Gemini API 完整讲解 · 文本生成

写给懂工程、懂 HTTP/SDK 的程序员。以官方 google-genai Python SDK 为主线，cURL/REST 对照。聚焦经典 generateContent 接口。

时效与来源说明（务必先读）。本文核对日期 2026-06-30。事实来自：Gemini API 的 v1beta machine-readable discovery 文档（rev 20260629，权威）、官方 googleapis/python-genai SDK 源码与 google-gemini/cookbook，以及对 ai.google.dev 官方页的搜索摘要。编写时本环境出口代理封锁了对 ai.google.dev 的直接抓取，故凡涉及易变的精确数字（定价、各模型上下文/输出上限、限流配额、thinkingBudget 各档范围），请以官方文档与控制台为准——文中已用 ⚠️ 标注此类项。此外，字段名 / 枚举值 / 异常类已额外用官方 google-genai SDK v2.10.0 源码（PyPI 直取）逐一核实，此类以 ✅ 标注、可放心使用。

1概览与心智模型

1.1 2026 年的格局：两套生成接口并存

调用 Gemini 生成内容，目前有两条 API 表面：

  • 经典 generateContent / streamGenerateContent（文档中标为 "Generate Content API (Legacy)"）。用 contents/parts 结构，返回 candidates，函数调用走 functionCall/functionResponse part，接地元数据在 candidates[].groundingMetadata。官方明确：它仍是生产负载的推荐路径、仍受支持。
  • 新的 Interactions API（client.interactions.create(...)，REST 路径不同），面向 agentic / 服务端状态管理，是新项目的推荐接口。但它尚未暴露接地引用元数据等部分能力。

本指南聚焦经典 generateContent——它字段稳定、SDK 支持完整、是绝大多数现有生产代码所用。需要接地引用、要渲染搜索建议时也只能用它。涉及差异处会点出 Interactions API。

1.2 无状态

经典接口和 Claude/OpenAI 一样无状态：服务端不保存对话，每次请求都要带全量历史（contents 数组）。所谓"多轮"是你在客户端不断追加（见第 6 节）。Interactions API 则改为服务端维护历史（previous_interaction_id）。

1.3 当前模型阵容（2026 年中）

Gemini 已演进到 3.x / 3.5 系列，2.5 系列正在退役、2.0 系列已关停。

1.4 与 Claude/OpenAI 的几个心智差异

  • 认证用 x-goog-api-key 头（不是 x-api-key / Authorization: Bearer）。
  • API 版本在 URL 路径段（v1beta），没有日期式版本请求头。
  • 助手角色叫 model（不是 assistant）。
  • 方法名是动词冒号式：models/{model}:generateContent。

2认证、SDK 与端点

2.1 官方 SDK：google-genai

官方 Python SDK 是 google-genai（from google import genai），2025 年 5 月 GA。旧的 google-generativeai（import google.generativeai）已弃用（弃用生效 2025-11-30），不再获得新功能——新代码一律用 google-genai。

安装 + 最简调用
# pip install -U google-genai
from google import genai

# 自动读取环境变量 GEMINI_API_KEY（也接受 GOOGLE_API_KEY）
client = genai.Client()
# 或显式： client = genai.Client(api_key="YOUR_API_KEY")

resp = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="用一句话解释 CAP 定理",
)
print(resp.text)   # 便捷属性：拼接所有文本 part

3请求结构详解

顶层字段：contents（必填）、systemInstruction、generationConfig、safetySettings、tools/toolConfig。

3.1 contents / role / parts

  • contents 是数组，每个元素是一个 Content：role 为 "user" 或 "model"（单轮可省略 role），parts 是数组。
  • 一个 part 通常是 {"text": "..."}，也可是 inlineData/fileData（多模态，第 12 节）、functionCall/functionResponse（第 9 节）等。
  • 系统指令不放在 contents 里，而是顶层 systemInstruction。

3.2 systemInstruction

系统指令
from google.genai import types

resp = client.models.generate_content(
    model="gemini-3.5-flash",
    contents="如何安全地读一个大 JSON 文件？",
    config=types.GenerateContentConfig(
        system_instruction="你是一个严谨的后端工程助手，尽量给可运行示例。",
    ),
)
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" -H 'Content-Type: application/json' -X POST -d '{
    "systemInstruction": { "parts": [ { "text": "你是一个严谨的后端工程助手。" } ] },
    "contents": [ { "parts": [ { "text": "如何安全地读一个大 JSON 文件？" } ] } ]
  }'

3.3 generationConfig 字段

字段（REST / SDK）范围 / 默认说明
temperature0.0–2.0 ⚠️（旧文档曾写 0.0–1.0，随模型不同）采样温度
topP / top_p0.0–1.0核采样
topK / top_k≥1（部分较新模型已忽略 ⚠️）Top-K 采样
maxOutputTokens上限=模型最大输出输出 token 上限
stopSequences最多 5 个命中即停（不含在输出中）
candidateCount1–8，默认 1返回候选数
responseMimeType / responseSchema—结构化输出（第 8 节）
thinkingConfig—推理控制（第 7 节）

⚠️ temperature 上限（1.0 还是 2.0）随模型不同，是文档间最易冲突处；部分较新模型不再支持 topK。逐模型核对。

SDK 里这些参数都通过 config=types.GenerateContentConfig(...) 传入（旧 SDK 是 generation_config=）。

4响应结构与 finishReason

结构
{
  "candidates": [
    {
      "content": { "role": "model", "parts": [ { "text": "..." } ] },
      "finishReason": "STOP",
      "safetyRatings": [ /* category + probability */ ],
      "index": 0
    }
  ],
  "usageMetadata": {
    "promptTokenCount": 10,
    "candidatesTokenCount": 25,
    "totalTokenCount": 35,
    "thoughtsTokenCount": 0      // thinking 模型才有
  },
  "modelVersion": "gemini-3.5-flash"
}

SDK 里 resp.text 便捷拼接所有文本 part；更底层是 resp.candidates[0].content.parts[*]。访问前注意可能有非文本 part（thought、functionCall 等）。

4.1 finishReason 取值

值含义
STOP正常结束（偶有 STOP 但 parts 为空的边界情况）
MAX_TOKENS达到 maxOutputTokens（thinking 模型可能思考耗尽 token 而文本为空）
SAFETY安全策略拦截（看 safetyRatings）
RECITATION疑似复述受保护内容
OTHER其他边界情况

✅ SDK 源码核实的完整枚举（FinishReason，v2.10.0）：STOP、MAX_TOKENS、SAFETY、RECITATION、LANGUAGE、OTHER、BLOCKLIST、PROHIBITED_CONTENT、SPII、MALFORMED_FUNCTION_CALL、UNEXPECTED_TOOL_CALL、FINISH_REASON_UNSPECIFIED，以及图像相关的 IMAGE_SAFETY / IMAGE_PROHIBITED_CONTENT / IMAGE_RECITATION / IMAGE_OTHER / NO_IMAGE。

usageMetadata 关键字段：promptTokenCount（输入）、candidatesTokenCount（输出）、totalTokenCount、thoughtsTokenCount（思考）、缓存场景的 cachedContentTokenCount。

6多轮对话

multi-turn
from google.genai import types

chat = client.chats.create(
    model="gemini-3.5-flash",
    history=[
        types.Content(role="user",  parts=[types.Part(text="你好")]),
        types.Content(role="model", parts=[types.Part(text="你好！有什么可以帮你？")]),
    ],
)
print(chat.send_message("我家有两只狗。").text)
print(chat.send_message("那我家一共有几只爪子？").text)   # 会记得"两只狗"
# chat.get_history() 取历史；chat.send_message_stream(...) 流式
# REST 无 chat 概念：把每轮追加进 contents，user / model 交替
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" -H 'Content-Type: application/json' -X POST -d '{
    "contents": [
      { "role": "user",  "parts": [ { "text": "你好，我家有两只狗。" } ] },
      { "role": "model", "parts": [ { "text": "好的，两只狗。" } ] },
      { "role": "user",  "parts": [ { "text": "那我家一共有几只爪子？" } ] }
    ]
  }'

新的 Interactions API 改为服务端维护历史（传 previous_interaction_id 而非全量 contents），这是它与经典接口的主要差异之一。

7Thinking / 推理

Gemini 2.5 与 3.x 都是"思考模型"：给出答案前先生成内部推理 token。这些 token 按输出计费，并单独报在 thoughtsTokenCount。

7.1 两套控制：thinkingBudget（2.5）vs thinkingLevel（3.x）

通过 generationConfig.thinkingConfig（SDK：types.ThinkingConfig）控制。字段（核对自官方 SDK 源码）：

字段（SDK / REST）类型说明
thinking_budget / thinkingBudgetint0 = 关闭，-1 = 自动（动态）；范围/默认随模型而定。Gemini 2.5 用它。
thinking_level / thinkingLevelenumMINIMAL / LOW / MEDIUM / HIGH。Gemini 3.x 用它（思考常开，按档调）。
include_thoughts / includeThoughtsbooltrue 时返回思考摘要 part

两者互斥/分代：2.5 系列用 thinkingBudget，3.x 用 thinkingLevel。把 thinkingLevel 用在 2.5/Flash-Lite 上会报 thinking_level not supported。

7.2 thinkingBudget 各档（Gemini 2.5）

模型范围 ⚠️默认0 可关闭？
2.5 Pro128 – 32768动态(-1)否（不能关，下限 128）
2.5 Flash0 – 24576动态(-1)是
2.5 Flash-Lite512 – 24576默认关闭是（需 -1 显式开启动态）

✅ 字段核实自 SDK v2.10.0 ThinkingConfig：include_thoughts(bool)、thinking_budget(int，docstring 原文 "0 is DISABLED. -1 is AUTOMATIC. default values and allowed ranges are model dependent")、thinking_level(枚举 MINIMAL/LOW/MEDIUM/HIGH)。⚠️ 但上表各档具体数字（128/24576/32768…）SDK 不含（它明说"因模型而异"），来自搜索摘要，精确范围以 官方 thinking 页 为准；设超范围值时 API 就近调整。

7.3 取思考摘要与示例

thinking
from google.genai import types

resp = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="逐步解释天空为何是蓝色的。",
    config=types.GenerateContentConfig(
        thinking_config=types.ThinkingConfig(
            thinking_budget=4096,    # 0=关闭(Flash系)；-1=动态
            include_thoughts=True,   # 返回思考摘要
        ),
    ),
)
for part in resp.candidates[0].content.parts:
    if not part.text:
        continue
    if part.thought:                       # thought=True 即思考摘要
        print("[思考摘要]", part.text)
    else:
        print("[答案]", part.text)
print("思考 token:", resp.usage_metadata.thoughts_token_count)
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" -H 'Content-Type: application/json' -d '{
    "contents": [ { "parts": [ { "text": "逐步解释天空为何是蓝色的。" } ] } ],
    "generationConfig": {
      "thinkingConfig": { "includeThoughts": true, "thinkingBudget": 4096 }
    }
  }'
# 响应里思考 part 形如 {"thought": true, "text": "..."}；usageMetadata.thoughtsTokenCount 计思考

多轮/函数调用时，part 上的 thoughtSignature（不透明签名）用于跨轮保留推理上下文，应原样回填。

8结构化输出（JSON）

强制 JSON 输出靠 generationConfig 里两个字段配合：

  • responseMimeType：application/json（或 text/x.enum 见下），默认 text/plain。
  • responseSchema：OpenAPI 3.0 子集 schema。设了它就必须把 responseMimeType 设为 application/json。

8.1 Python：Pydantic / TypedDict / Enum 都支持

SDK 接受 Pydantic BaseModel、list[Model]、TypedDict、Enum、或原始 dict/Schema 作为 response_schema，并把校验后的对象放进 response.parsed。

structured output
from google import genai
from pydantic import BaseModel

client = genai.Client()

class Ingredient(BaseModel):
    name: str
    quantity: str

class Recipe(BaseModel):
    recipe_name: str
    ingredients: list[Ingredient]

resp = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="列举几个常见饼干配方。",
    config={
        "response_mime_type": "application/json",
        "response_schema": list[Recipe],
    },
)
recipes: list[Recipe] = resp.parsed   # 已校验的 Pydantic 对象
print(resp.text)                       # 原始 JSON 字符串

8.2 responseSchema vs responseJsonSchema

另有一个较新字段 responseJsonSchema，接受标准 JSON Schema（支持 $ref/$defs/oneOf/prefixItems 等）。

responseSchemaresponseJsonSchema
方言OpenAPI 3.0 子集标准 JSON Schema
互斥二选一，不可同设；两者都要求 responseMimeType: application/json
何时用默认；Pydantic/TypedDict 便利原生 JSON Schema，或前者处理不对时回退

propertyOrdering（非标准扩展）可控制生成 JSON 的属性顺序——模型对顺序敏感，schema 与示例顺序不一致会损质量。

8.3 枚举模式

enum
import enum
class Instrument(enum.Enum):
    PERCUSSION = "Percussion"; STRING = "String"
    WOODWIND = "Woodwind"; BRASS = "Brass"; KEYBOARD = "Keyboard"

resp = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="双簧管属于哪类乐器？",
    config={
        "response_mime_type": "text/x.enum",   # 返回单个枚举串，无 JSON 引号

print(resp.parsed)    # Instrument.WOODWIND

  -H "x-goog-api-key: $GEMINI_API_KEY" -H 'Content-Type: application/json' -d '{
    "contents": [{ "parts": [{ "text": "判断情感。" }] }],
    "generationConfig": {
      "responseMimeType": "application/json",
      "responseJsonSchema": {

        "required": ["sentiment"]

⚠️ text/x.enum 由 SDK 解析逻辑与搜索摘要佐证，未从官方页逐字确认；字段名（responseMimeType/responseSchema/responseJsonSchema/propertyOrdering）核对自官方 SDK 源码（可靠）。

9函数调用

模型只请求调用你声明的函数（返回 functionCall part），由你执行并回填 functionResponse。字段核对自 v1beta discovery 文档（rev 20260629）。

9.1 声明与配置

  • tools[].functionDeclarations[]：每个含 name / description / parameters（OpenAPI 子集 Schema）或 parametersJsonSchema（原始 JSON Schema）。
  • toolConfig.functionCallingConfig.mode：AUTO（默认）/ ANY（强制至少调一个）/ NONE（禁止）/ VALIDATED（较新）；配 allowedFunctionNames 限定可调函数。
  • 并行调用：单次响应可含多个 functionCall part；Gemini 3 起每个 functionCall 带唯一 id，回填 functionResponse 时应回显同一 id。

9.2 完整手动循环

function calling
from google import genai
from google.genai import types

client = genai.Client()

function = types.FunctionDeclaration(
    name='get_current_weather',
    description='查询某地当前天气',
    parameters_json_schema={
        'type': 'object',
        'properties': {'location': {'type': 'string', 'description': '城市，如 Boston, MA'}},
        'required': ['location'],
    },
)
tool = types.Tool(function_declarations=[function])

def get_current_weather(location: str) -> str:
    return f"{location} 当前 21°C，晴。"

user_turn = types.Content(role='user', parts=[types.Part.from_text(text='波士顿天气如何？')])
resp = client.models.generate_content(
    model='gemini-3.5-flash', contents=[user_turn],
    config=types.GenerateContentConfig(tools=[tool]),
)

contents = [user_turn]
while resp.function_calls:                          # 可能并行多个
    contents.append(resp.candidates[0].content)       # 原样回填 model 轮
    tool_parts = []
    for fc in resp.function_calls:
        try:
            payload = {'result': get_current_weather(**fc.args)}
        except Exception as e:
            payload = {'error': str(e)}
        tool_parts.append(types.Part.from_function_response(name=fc.name, response=payload))
    contents.append(types.Content(role='tool', parts=tool_parts))
    resp = client.models.generate_content(
        model='gemini-3.5-flash', contents=contents,
        config=types.GenerateContentConfig(tools=[tool]),
    )

print(resp.text)
# 第一步：声明工具
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent" \
  -H "x-goog-api-key: $GEMINI_API_KEY" -H 'Content-Type: application/json' -X POST -d '{
    "contents": [{"role":"user","parts":[{"text":"波士顿天气如何？"}]}],
    "tools": [{"functionDeclarations":[{
      "name":"get_current_weather",
      "description":"查询某地当前天气",
      "parameters":{"type":"object","properties":{"location":{"type":"string"}},"required":["location"]}
    }]}],
    "toolConfig": {"functionCallingConfig":{"mode":"ANY","allowedFunctionNames":["get_current_weather"]}}
  }'
# 模型返回 part: {"functionCall":{"name":"get_current_weather","args":{"location":"Boston, MA"},"id":"fc_abc"}}

# 第二步：把历史 + functionResponse 一起回传（回显 id）
# contents 追加 {"role":"model",...functionCall...} 和
#   {"role":"tool","parts":[{"functionResponse":{"name":"get_current_weather","id":"fc_abc","response":{"result":"21C 晴"}}}]}

若直接把 Python 可调用对象传进 tools=[get_current_weather] 且不关闭自动函数调用，SDK 会替你跑完整循环。要手动控制就传 automatic_function_calling=types.AutomaticFunctionCallingConfig(disable=True)。

各模态的 token 成本与上限

模态token 成本 ⚠️上限 / 备注 ⚠️
图片随分辨率/分块而定png/jpeg/webp/heic/heif

File API 要点

  • ✅ 上传 client.files.upload(file=...) 返回 File（SDK 核实字段：name/uri/mime_type/size_bytes/state/source/expiration_time/download_uri/error）；FileState ∈ PROCESSING/ACTIVE/FAILED，大文件/视频异步处理，用前等 state == "ACTIVE"。
  • ⚠️ 限制（SDK 不含、文档级）：单文件约 2GB、单项目约 20GB；保存 48 小时后自动删除（见 expiration_time）。
  • 操作：files.list() / files.get() / files.delete(name=...)。

⚠️ part 字段结构核对自 discovery doc（可靠）；但各模态 token 单价、时长/页数/大小上限、48 小时保存期、2GB/20GB 配额来自搜索摘要。尤其注意：内联数据上限历史为 20MB，2025 年官方博客宣布升到 100MB，不同官方页可能仍写 20MB——超过就改用 File API。具体数字请以官方 Files 及各模态页为准。

13上下文缓存

大段前缀（长文档、长系统指令、few-shot）在多次请求中复用时，缓存能让命中部分大幅降价（响应里见 cachedContentTokenCount）。Gemini 有两种缓存模式：

模式开启方式存储费最小 token 门槛 ⚠️
隐式缓存2.5+ 默认开启，无需写代码；前缀命中自动降价无2.5 Flash 约 1,024 / 2.5 Pro 约 2,048
显式缓存手动建 CachedContent（client.caches.create()），保证折扣有（按 token×时长）约 32,768（历史曾为 4,096，需核对）

✅ 字段核实自 SDK v2.10.0。建缓存 CreateCachedContentConfig：contents、system_instruction、tools/tool_config、ttl（时长串，默认 1 小时）或 expire_time（绝对时间）、display_name、kms_key_name。缓存对象 CachedContent：name、model、expire_time、usage_metadata。请求里用 GenerateContentConfig.cached_content（= 缓存 name）引用（REST 为顶层 cachedContent）。⚠️ 缓存命中约 9 折优惠、门槛与存储单价 SDK 不含，以官方页为准。

context caching
from google.genai import types

cache = client.caches.create(
    model="gemini-2.5-flash",
    config=types.CreateCachedContentConfig(
        system_instruction="你是这份大文档的专家。",
        contents=[big_document],          # 要缓存的稳定前缀
        ttl="3600s",
    ),
)
resp = client.models.generate_content(
    model="gemini-2.5-flash",
    contents="总结要点",
    config=types.GenerateContentConfig(cached_content=cache.name),
)
print(resp.usage_metadata.cached_content_token_count)  # 命中的缓存 token

# 管理： client.caches.list() / update(name, config=...ttl) / delete(name=...)

多数场景无需显式缓存——2.5+ 的隐式缓存已自动给前缀命中降价且无存储费。只有需要"保证命中折扣"或缓存很大且高频复用时才手动建 CachedContent。把稳定前缀放在最前、易变内容放最后，命中率最高。

⚠️ 字段名/方法签名核对自 v1beta discovery doc（可靠）；但显式缓存最小 token 门槛（32,768 vs 历史 4,096）、9 折优惠、存储单价来自搜索摘要/博客，请以官方 Caching 与 Pricing 页为准。

16错误处理与限流

Gemini 走 Google API 的标准错误模型：HTTP 状态码 + error.status（如 INVALID_ARGUMENT）+ error.message。

HTTPstatus常见原因可重试
400INVALID_ARGUMENT / FAILED_PRECONDITION请求体/参数非法；或免费层未启用计费否
403PERMISSION_DENIEDAPI key 无效/无权限否
404NOT_FOUNDmodel id 拼错 / 资源不存在否
429RESOURCE_EXHAUSTED超出 RPM/TPM/RPD 配额是（退避重试）
500INTERNAL服务端错误是
503UNAVAILABLE服务暂时过载是
504DEADLINE_EXCEEDED处理超时（输入过大等）是（缩小输入）

限流三个维度（每项目、每模型，撞任一即 429）：RPM（每分钟请求）/ TPM（每分钟 token）/ RPD（每天请求）。配额按付费层级递增：Free（最低，部分地区不可用）→ Tier 1（启用计费即刻生效）→ Tier 2（约累计消费 $250 且首次付款满 30 天 ⚠️）→ Tier 3（更高门槛，常见 $1,000+ ⚠️）。

分类：不可重试 = 400/403/404；可重试 = 429/500/503/504。429 用带 jitter 的指数退避。⚠️ 各模型精确 RPM/TPM/RPD 与 Tier 2/3 门槛经常变动、且存在文档外的后端限流（有人在文档配额内仍遇 429），具体以官方 Rate limits 页 + 控制台为准，错误语义见 Troubleshooting。

retry
import time, random
from google.genai import errors

def call_with_retry(fn, max_retries=5):
    for i in range(max_retries):
        try:
            return fn()
        except errors.APIError as e:
            if e.code in (429, 500, 503) and i < max_retries - 1:
                time.sleep(min(2 ** i + random.random(), 60))
                continue
            raise

✅ 核实自 SDK v2.10.0 google.genai.errors：基类 APIError（属性 .code / .status / .message），子类 ClientError(4xx) 与 ServerError(5xx)；另有 UnknownFunctionCallArgumentError 等函数调用相关的 ValueError。上例的 errors.APIError + e.code 即据此。
